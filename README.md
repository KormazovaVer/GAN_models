### Папка GAN_models содержит 4 ноутбука с реализованной архитектурой CycleGAN для генерации картин в стиле Клода Моне и Пабло Пикассо:   

1. Модель CycleGAN для генерации картин в стиле Моне + веса к ней.
2. Модель CycleGAN для генерации картин в стиле Пикассо + веса к ней.
3. Дообученная модель CycleGAN для генерации картин в стиле Моне + веса к ней.
4. Модель CycleGAN  с фильтрами повышения резкости для генерации картин в стиле Моне + веса к ней.


Этот код реализует цикл обучения для модели генеративно-состязательной сети (GAN), которая предназначена для преобразования
фотографий в картины в стиле Клода Моне (Пабло Пикассо). Основные компоненты кода и архитектуры GAN описаны ниже.


### Основные компоненты кода:


#### Импорт библиотек:
Используются библиотеки PyTorch для построения и обучения нейронных сетей, а также библиотеки для обработки изображений и аугментации данных.


#### Загрузка данных:
Датасеты изображений картин Моне (Пикассо) и фотографий загружаются из Google Drive и распаковываются.


#### Определение классов:
MonetDataset: пользовательский класс, который наследует Dataset из PyTorch. Он отвечает за загрузку изображений и их аугментацию.


#### Архитектура GAN:

#### Дискриминатор (Discriminator):
Сеть, которая определяет, является ли изображение реальным (изображение Моне (Пикассо) или фотография) или поддельным
(сгенерированным генератором). Состоит из нескольких сверточных слоев, которые последовательно уменьшают пространственное разрешение изображения 
и увеличивают количество каналов. Использует Leaky ReLU в качестве функции активации и Instance Normalization для стабилизации обучения.   

#### Генератор (Generator):
Сеть, которая генерирует изображения в стиле Моне (Пикассо) на основе входных фотографий. Начинается с нескольких сверточных слоев, 
за которыми следуют остаточные блоки, которые помогают сохранить информацию о входном изображении. Затем используются транспонированные свертки для
увеличения разрешения изображения до исходного размера. В качестве активации используется функция tanh.   


#### Обучение:
В функции train_fn происходит обучение дискриминатора и генератора. Дискриминатор обучается отличать реальные изображения от поддельных, а генератор
обучается создавать изображения, которые будут классифицированы дискриминатором как реальные.


#### Потери:
Используются различные функции потерь, включая:  
Adversarial Loss: для оценки качества сгенерированных изображений.  
Cycle Loss: для обеспечения сохранения информации о входном изображении в процессе преобразования.  
Identity Loss: для сохранения информации о входном изображении.   


#### Вычисление значения FID (Fréchet Inception Distance):
В этом разделе вычисляется значение FID между реальными изображениями и сгенерированными. FID используется для оценки качества генерации изображений: 
чем ниже значение, тем ближе сгенерированные изображения к реальным. Для этого используется функция calculate_fid_given_paths, которая принимает пути 
к каталогам с реальными и сгенерированными изображениями и возвращает значение FID.


#### Тестирование модели:
Раздел включает функцию test_model, которая загружает тестовое изображение, генерирует его версии в стиле Моне (Пикассо) и фотографическом стиле с помощью генераторов,
а затем визуализирует оригинал и результаты генерации. Также предусмотрена обработка ошибок при загрузке весов модели, чтобы обеспечить корректную работу тестирования.  


#### Заключение:
Этот код представляет собой классическую реализацию GAN с использованием архитектуры U-Net для генератора и стандартного дискриминатора, который хорошо подходит для задач
преобразования стиля. Обучение происходит итеративно, где обе сети (дискриминатор и генератор) конкурируют друг с другом, что приводит к улучшению качества сгенерированных изображений.
